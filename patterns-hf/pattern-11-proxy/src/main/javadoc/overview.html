Прокси-класс — это некоторая «надстройка» над оригинальным классом, которая позволяет нам при необходимости изменить его поведение. К примеру, вне зависимости от того, какой именно метод будет вызван у нашего объекта, мы хотим, чтобы при вызове любого из них печаталось «Привет!» Есть несколько самых популярных кейсов использования Прокси-объектов (Заместителей): - Удалённый заместитель (для RMI): управляет доступом к удалённому ресурсу. Этот прокси объект необходим, т.к. перед непосредственно вызовом нужного метода объекта нужно произвести предварительные действия такие как принятие по сети информацию об удалённом вызове, произвести десереализацию параметров метода,а уж потом вызвать нужный метод т.е. перед неосредственно вызовом нужного метода, нужно произвести предварительные действия - Виртуальный заместитель: управляет доступом к "тяжёлому" ресурсу, т.е. создавать его только при необходимости. - Защитный заместитель: контролирует доступ к ресурсу. Его можно сделать с помощью средств Java API (см. класс Proxy - динамический прокси) Например, когда нужно, перед вызовом методов проверить авторизацию или ограничить вызов каких-то медов, например, в объекте Person есть метод setSelary, но назначать зарплату может только руководитель, а не а бы кто.
