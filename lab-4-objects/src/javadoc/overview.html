Заметки для OCA/OCP. 1. Import Statements - нужен только для удобства, т.к. позволяет указывать более короткие имено классов, не указывая полное имя пакета. В отличие от #include в С в Java Import не обязателен, т.к. компилятор С++ не просматривает файлы кроме компилируемого и указанных в #include. Загрузчик классов Java ищет классы не по import (например по classpass и пр.) 2. Static Import Statements - позволяет импортировать не только классы, но и статические поля (константы) и методы import static java.lang.System.out; // теперь можно писать так: out.println(...); 3. Импортировать можно пакеты, классы, методы, НО нельзя импортировать, например, import java.*; (!) Т.е. более широкий поиск, т.е. поиск по нескольким пакетам, использовать нельзя 4. Класс может быть помечен модификаторами: - доступа (видимости). Их 4: public, protected, private, null (dafault) - см. lab-5-6 - Nonaccess modifiers: strictfp, final, abstract (см. lab-5-6 class Person) - идут в дополнение * strictfp, его цель — определить точное и единообразное выполнение операции над числами с плавающей запятой на всех JVM (соответствие с IEEE 754 standard rules for floating points) используется для классов, методов, но не для переменных! * единовременно нельзя использовать модификаторы final и abstract
