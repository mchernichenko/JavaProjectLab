Заметки для OCA/OCP. 1. Import Statements - нужен только для удобства, т.к. позволяет указывать более короткие имено классов, не указывая полное имя пакета. В отличие от #include в С в Java Import не обязателен, т.к. компилятор С++ не просматривает файлы кроме компилируемого и указанных в #include. Загрузчик классов Java ищет классы не по import (например по classpass и пр.) 2. Static Import Statements - позволяет импортировать не только классы, но и статические поля (константы) и методы import static java.lang.System.out; // теперь можно писать так: out.println(...); 3. Импортировать можно пакеты, классы, методы, НО нельзя импортировать, например, import java.*; (!) Т.е. более широкий поиск, т.е. поиск по нескольким пакетам, использовать нельзя 4. Классы,методы,переменные могут быть помечены модификаторами: - доступа (видимости). Их 4: public, protected, private, null (default) - см. lab-5-6 - Nonaccess modifiers: strictfp, final, abstract (см. lab-5-6 class Person) - идут в дополнение ! Важно: классы могут иметь только 2 уровеня доступа: public и default, а методы и переменные класса все четыре Локальные переменные не могут иметь модификатьров доступа (только модификатор final) Более побробно о модификаторах см. в lab-5-6 (особенно PGN в javadoc) Всегда сначала нужно смотреть на уровень доступа у класса. Если один класс не доступен для другого, то и методы и переменные тоже не доступны вне зависимости от их уровня доступа. * strictfp, его цель — определить точное и единообразное выполнение операции над числами с плавающей запятой на всех JVM (соответствие с IEEE 754 standard rules for floating points) используется для классов, методов, но не для переменных! * единовременно нельзя использовать модификаторы: -final и abstract, - а также final и private и - и abstract и static 5. final - предостварает какое либо изменение метода или переменной => насленование final метода невозможно.
